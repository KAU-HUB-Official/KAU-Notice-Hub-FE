# 공지 분류 체계

## 목적
크롤링 범위가 넓어지면서 `source_name`을 그대로 나열하는 방식만으로는 탐색성이 떨어집니다.
현재 MVP는 공지를 먼저 사용 대상자 관점으로 묶고, 필요한 경우에만 중분류 또는 세부 홈페이지 필터를 제공합니다.

## 필터 구조
기본 탐색 흐름은 아래 순서입니다.

1. 대상자 대분류 선택
2. 중분류 선택
3. 일부 대분류에서만 세부 홈페이지 선택
4. 검색어 입력

세부 홈페이지 필터는 모든 대분류에 노출하지 않습니다.
현재 세부 홈페이지 필터가 노출되는 대분류는 아래 3개입니다.

- `학부 재학생(학과/전공별)`
- `대학원생`
- `평생·전문교육원`

그 외 대분류는 중분류까지만 선택합니다.

## 대분류 목록
현재 대분류 노출 순서는 `src/lib/notices.ts`의 `AUDIENCE_GROUP_ORDER`를 따릅니다.

| 대분류 | 용도 |
| --- | --- |
| `전 구성원 공통` | 공식 홈페이지 공통 공지 |
| `학부 재학생(학과/전공별)` | 단과대, 학부, 학과, 전공 홈페이지 공지 |
| `신입생·저학년` | 새내기성공센터, 드림칼리지디자인 공지 |
| `재학생 비교과·글로벌 프로그램` | 국제교류, 비교과, 공모전, 대회, 산학/부트캠프, 교수학습 프로그램 |
| `취업·창업 준비생` | 취업/채용/창업 관련 공지 |
| `대학원생` | 일반대학원, 경영대학원, 항공우주정책대학원 등 대학원 계열 |
| `평생·전문교육원` | 평생교육원 및 항공 관련 전문교육원 |
| `그 외` | 생활관, 인권센터, 학술정보관, LMS, 입학처, 박물관 등 보조 분류 |

## 중분류와 세부 홈페이지
### 전 구성원 공통
중분류:
- `일반`
- `학사`
- `장학/대출`
- `입찰`
- `행사`

세부 홈페이지:
- 사용하지 않음

### 학부 재학생(학과/전공별)
중분류:
- `공과대`
- `AI융합대`
- `항공경영대`
- `그 외 학부`

세부 홈페이지:
- 각 단과대/학부/학과/전공 source를 그대로 노출
- 예: `컴퓨터공학과`, `신소재공학과 학부`, `항공우주공학과`, `항공교통물류학부`, `자유전공학부`

### 신입생·저학년
중분류:
- `새내기성공센터`
- `드림칼리지디자인`

세부 홈페이지:
- 사용하지 않음

### 재학생 비교과·글로벌 프로그램
중분류:
- `국제교류`
- `첨단분야 부트캠프`
- `산학협력`
- `교수학습센터`

세부 홈페이지:
- 사용하지 않음

### 취업·창업 준비생
중분류:
- `대학일자리플러스센터`
- `학과 취업공지`

세부 홈페이지:
- 사용하지 않음

### 대학원생
중분류:
- 사용하지 않음

세부 홈페이지:
- `한국항공대학교 대학원`
- `한국항공대학교 경영대학원`
- `한국항공대학교 항공우주정책대학원`
- `한국항공대학교 신소재공학과 대학원`

### 평생·전문교육원
중분류:
- 사용하지 않음

세부 홈페이지:
- `한국항공대학교 평생교육원`
- `한국항공대학교 비행교육원`
- `한국항공대학교 항공교통관제교육원`
- `한국항공대학교 항공기술교육원`
- `한국항공대학교 항공안전교육원`

## 주요 분류 규칙
분류는 `source_name` 또는 정규화된 `sources` 값을 기준으로 수행합니다.
핵심 구현은 `src/lib/notices.ts`에 있습니다.

| source 키워드 | 대분류 | 중분류/세부 처리 |
| --- | --- | --- |
| `공식 홈페이지` | `전 구성원 공통` | 제목/분류값 기준으로 `일반`, `학사`, `장학/대출`, `입찰`, `행사` 분류 |
| `학과`, `학부`, `전공`, 단과대 키워드 | `학부 재학생(학과/전공별)` | 단과대 중분류 + 세부 홈페이지 필터 |

## UI 동작
- 대분류는 `AudienceNav` 탭으로 선택합니다.
- 중분류는 `SourceGroupFilter` 탭으로 단일 선택합니다.
- 선택된 중분류를 다시 클릭하면 `전체 중분류` 상태로 해제됩니다.
- 세부 홈페이지는 `SourceNav` 탭으로 단일 선택합니다.
- 세부 홈페이지를 다시 클릭하면 `전체 홈페이지` 상태로 해제됩니다.
- 대분류를 변경하면 중분류와 세부 홈페이지 선택은 초기화됩니다.
- 중분류를 변경하면 세부 홈페이지 선택은 초기화됩니다.
- 세부 홈페이지 필터가 필요 없는 대분류에서는 `SourceNav`를 렌더링하지 않습니다.

## API 동작
목록 API:

```text
GET /api/notices?audience=...&group=...&source=...&q=...
```

쿼리 파라미터:
- `audience`: 대분류
- `group`: 중분류
- `source`: 세부 홈페이지
- `q`: 검색어
- `page`: 페이지
- `pageSize`: 페이지 크기

`source` 필터는 아래 대분류에서만 적용됩니다.

- `학부 재학생(학과/전공별)`
- `대학원생`
- `평생·전문교육원`

다른 대분류에서 `source` 쿼리가 들어오면 서버에서 무시합니다.
중분류가 없는 대분류에서 `group` 쿼리가 들어와도 서버에서 무시합니다.

## 데이터 정규화
입력 JSON의 `source_name`은 문자열 또는 문자열 배열일 수 있습니다.
정규화 과정에서 다음 필드를 사용합니다.

- `source`: 대표 source
- `sources`: 배열 형태로 보존한 전체 source 목록

`source_name` 배열에 여러 홈페이지가 들어 있는 공지는 각 홈페이지 필터에 모두 걸릴 수 있습니다.

## 유지보수 원칙
- `기타`라는 이름 대신 현재는 `그 외`를 사용합니다.
- 대분류 하나에 source가 1개뿐이면 별도 대분류로 만들기보다 주변 맥락에 맞춰 군집화합니다.
- 대분류 안의 중분류가 다시 같은 이름을 반복하면 중분류를 제거하고 세부 홈페이지 필터로 처리합니다.
- 세부 홈페이지 필터는 사용자가 실제로 홈페이지 단위로 좁혀볼 필요가 있는 대분류에만 노출합니다.
- 새 source가 추가되면 먼저 `source_name`을 확인하고, `src/lib/notices.ts`의 `classifySourceToAudience`, `classifyNoticeSourceGroups`, `SOURCE_GROUP_ORDER`를 함께 점검합니다.
